构建“用户管理系统”后端 API

1. 使用 Spring Initializr 搭建项目。编写一套完整的 CRUD 接口，并确保 JSON 格式规范。
2. 使用 Validation 组件对 Controller 参数进行校验
3. 编写 GlobalExceptionHandler，实现全局异常拦截，不再向前端暴露堆栈信息。
4. 配置 application.yml 连接 MySQL。使用 MyBatis 完成对 User 数据的增删改查。
5. 分层架构：必须严格遵循 Controller -> Service -> Mapper 的三层架构，严禁在 Controller 中直接操作数据库。
6. 响应封装：所有 API 接口返回 Result<T> 对象（包含 status, msg, data），规范 API 交互。
7. 持久化：使用 MyBatis 实现用户信息落盘。
8. 在项目根目录使用 .gitignore 排除所有 target/ 和 .idea/ 文件夹。
